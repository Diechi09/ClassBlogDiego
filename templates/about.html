About Fantasy Forum

Fantasy Forum is a simple, community-driven space for NFL fantasy football discussion. It’s built to be fast, friendly, and focused on practical advice—whether you’re weighing a trade, scanning the waiver wire, or navigating injury chaos before kickoff.

Under the hood, this project is a full-stack Flask app with authentication, CRUD posts, comments, category flairs, and a small REST API. On the surface, it’s a place to learn from each other and win your league.

What you can do here

  • Post questions & takes with a flair: TRADE HELP, WAIVER WIRE ADVICE, INJURY TALK, OTHER.
  • Comment on threads to share insight, stats, and start/sit logic.
  • Edit or delete your own posts to keep information current.
  • Filter & browse by flair on the Home page to find the content you need fast.
  • Explore the API on the API Demo page, or pull JSON directly from endpoints like /api/posts.
  • Download data for research or auditing via Export All Posts (JSON).

How to get started

  1. Create an account (or log in if you already have one).
  2. Click New Post, choose a flair, and write your question or analysis.
  3. Use the Home page filters to find trade/waiver/injury topics relevant to you.
  4. Jump into the comments—share data, reasoning, and polite counter-points.

Community guidelines

  • Be constructive. Explain the “why” behind your advice.
  • Be respectful. No personal attacks or gatekeeping.
  • Add context. Scoring format, team needs, bye weeks, injuries matter.
  • Keep it relevant. Use flairs to help others discover your post.

Tech notes (for the curious)

Flask + SQLite + WTForms + Flask-Login power the site. The REST API supports JSON reads with search, pagination, and export. It’s intentionally lightweight so it’s easy to understand, extend, and deploy.
